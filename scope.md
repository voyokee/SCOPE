# BTC 中短期投资状态机 v3.3

总分 **100 分**。
用途：**中短期投资 / 波段交易 / 仓位管理**。
目标不是预测，而是：

* 约束在错误环境里做过大的动作
* 减少重复计分
* 提高同一套规则跨时间的一致性
* 让状态切换和仓位动作更稳定

分成三层：

* **周期层：25 分**
* **结构层：50 分**
* **脆弱性层：25 分**

---

# 零、统一判定优先级与通用阈值定义

这是 3.3 最重要的补丁之一。
它解决的问题是：**绝对阈值规则和分位数规则混用时，到底听谁的。**

## 0.1 判定优先级规则

**规则 1：已明确给出绝对阈值的指标，永远按绝对阈值打分。**
例如：

* Funding
* 年化 Basis
* 清算密集区距离现价
* OI 7 日增幅
* 状态切换分数阈值

**规则 2：未给出绝对阈值、仅使用“显著为正 / 明显为负 / 一般 / 明显改善 / 明显恶化”等描述时，才使用分位数标准。**

**规则 3：同一指标同一时点不得同时使用绝对阈值和分位阈值判分。**
若某指标主规则是绝对阈值，则分位数只能用于辅助描述，不能改变得分。

这条规则加入后，框架的一致性会明显提高。

---

## 0.2 分位数基准窗口

除特别说明外，所有分位数计算基于**90 日滚动窗口**。
变化项的“明显改善 / 明显恶化”基于**30 日变化分布**。

---

| 术语      | 量化定义                        |
| ------- | --------------------------- |
| 显著为正    | 该指标值处于近 90 日分布的 ≥75 分位      |

| 明显改善    | 该指标单日变化处于近 30 日变化分布的 ≥90 分位 |

### v3.3.1 补丁：渐进式 200D 距离评分 + 量价调节器

**问题**：v3.3 对 200D 做二元判定，跨越时评分跳变 3.5-6 分。且 50D 下方横盘期评分长期卡死。

| 距 200D 距离 | 基础分 |
| ---------- | --: |
| < 5%       |   7 |
| 5–10%      |   6 |
| 10–15%     |   5 |
| 15–20%     |   4 |
| > 20%      |   3 |

当价格 < 50D 且 < 200D 时：

| 距 200D 距离 | 基础分 |
| ---------- | --: |
| < 5%       |   2 |
| 5–15%      |   1 |
| > 15%      |   0 |

**补丁 B：量价调节器（±1 分）**

仅在中间地带（基础分 2–6）生效，根据量能结构调整：

| 条件 | 调节 |
| --- | --: |
| 5D/20D 量比 < 0.7 且 ATR 分位 < 25%（缩量+波动率压缩 = 积蓄突破） | +1 |
| 5D/20D 量比 > 1.3 且 7D 涨幅 > 0（放量上攻确认） | +1 |
| 5D/20D 量比 < 0.7 且 20D 斜率下行（缩量下行 = 衰竭） | -1 |
| 5D/20D 量比 > 1.3 且 7D 跌幅 > 3%（放量下跌 = 分布） | -1 |

调节后 clamp 到 [0, 10]。

## 2）OI 质量（7分）

这是 3.3 第二个关键补丁。
3.2 的问题不是逻辑错，而是**没有给 Funding 过热时的上限约束**。现在补上。

### 主打分规则

| 条件                    |  分数 |
| --------------------- | --: |
| 价格上涨，OI 7日增幅 < 10%    |   7 |
| 价格上涨，OI 7日增幅 10%–20%  | 5–6 |
| 价格上涨，OI 7日增幅 > 20%    | 2–4 |
| 价格停滞 / 下跌，但 OI 继续明显抬升 | 0–1 |

### Funding 上限修正条款

* 若 **Funding 绝对值 > 0.03%**，则 **OI 质量上限不得超过 5 分**
* 若 **Funding 绝对值 > 0.06%**，则 **OI 质量上限不得超过 3 分**

### 解释

这不是重复计分，而是**组合风控约束**。
OI 质量的主规则仍然看 OI 和价格的关系，但当 Funding 已经明显过热时，OI 再高分就不合理了。

### 3C）仓位一致性 / 供给扰动风险（4分）

这是 3.3 的第三个关键补丁。
不再叫“单边堆仓程度”，因为现在它包含的不只是堆仓，还包含供给扰动。

固定看四项，每满足一个“健康条件”得 1 分：

1. **25-delta skew 未极端偏斜**
   即：未达到近 90 日极端分位

2. **Long / Short Ratio 未出现明显单边**
   即：处于近 30 日分位的 20%–80% 区间

3. **Top Trader Positioning 未出现极端一致单边**
   即：净多/净空未达到近 30 日极端分位

4. **交易所 BTC 净流入未异常**
   即：3 日累计净流入未超过 30 日日均净流入的 2 倍

这是 3.3 最后一处修正。

### 触发条件

如果因降级触发减仓后，**下一个日线收盘**价格重新站上触发降级的关键位：

### 执行规则

* **可临时恢复至降级前仓位上限的 80%**
* **但状态标签不直接恢复**
* 状态标签仍需按正常升级规则确认

### 解释

快速回补恢复的是**仓位权限**，不是系统状态判断。
这样做能保留战术灵活性，同时避免状态标签被一根反抽 K 线轻易改写。

## 4）连续假信号熔断

如果 7 个自然日内触发快速回补 ≥ 2 次：

* 暂停快速回补规则 5 个交易日
* 期间所有状态切换严格按标准时间确认规则执行

目的：防止极端震荡期被反复折腾。

# 六、极端事件一票否决机制

以下情况**无论总分多少**，立即进入状态 D 并冻结 3 个日线收盘：

1. BTC 单日跌幅 > 15%
2. 主流交易所发生安全事件或暂停提币
3. 主要市场出台针对性监管禁令
4. 主流稳定币偏离锚定 > 2%

冻结期内不评分、不操作。冻结期结束后，按正常升级规则重新评估。

# 七、宏观脱敏规则

这一条保留 3.2 设计，不做改动。

### 触发条件

**BTC 相对强弱 ≥ 5/6 分，且连续 3 个日线收盘维持。**

### 效果

周期层得分设置**下限 = 10 分**。

### 退出条件

BTC 相对强弱连续 2 个日线收盘 < 4/6 时，取消下限保护，恢复实际得分。
